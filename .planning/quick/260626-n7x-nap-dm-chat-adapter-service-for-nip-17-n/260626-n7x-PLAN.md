# NAP-DM Chat Adapter Service

Implement a runtime-owned `dm` service for the unpublished NAP-DM wire surface.

Scope:
- Add local NAP-DM wire/result types because `@napplet/nap@0.21.0` npm latest has no `dm` subpath yet.
- Add `createDmService` that routes `dm.status`, `dm.conversations`, `dm.messages`, `dm.send`, `dm.subscribe`, and `dm.unsubscribe`.
- Add NIP-17 adapter using shell-owned ephemeral/host keys and relay pool publish/subscribe.
- Add NDR and Cordn adapters as host-injected runtime adapters so Kehto does not copy the active Hyprgate spike or vendor unstable protocol code.
- Add focused unit/e2e-style relay tests with ephemeral keys and an in-memory relay pool.
- Update exports, docs, changeset, and run build/test gates before PR.

Verification target:
- Focused Vitest for NAP-DM service/adapters.
- `pnpm --filter @kehto/services build`
- `pnpm type-check`
- broader gates before shipping.

Completed:
- Added runtime-owned `dm` service dispatch, ACL capabilities, shell advertisement, docs, and changeset.
- Added NIP-17, NDR, and Cordn adapters with local NAP-DM types until upstream peer packages publish the DM contract.
- Added local relay proof using `nak serve` with ephemeral NIP-17 keys.
