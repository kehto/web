# Repair v1.34 Release Workflow After Partial Publish

## Scope

The `v1.34` Release run passed build, type-check, unit tests, and E2E, then failed in publication:

- npm published `@kehto/nip@0.4.0`, `@kehto/services@0.11.1`, and `@kehto/shell@0.13.0`.
- npm rejected `@kehto/cli@0.2.0` and `@kehto/paja@0.2.0` with 404 / missing permission.
- JSR published `@kehto/nip@0.4.0`, `@kehto/services@0.11.1`, and `@kehto/shell@0.13.0`, then failed because `packages/paja` had no `jsr.json`.

## Plan

1. Add JSR manifests for `@kehto/paja` and `@kehto/cli`.
2. Run the existing JSR version sync so committed manifests match package versions after the v1.34 release PR.
3. Add a unit guard proving every public `packages/*` `@kehto/*` package has a `jsr.json`.
4. Verify locally with build, type-check, full unit tests, docs, JSR dry-run, and registry checks.

## Out Of Scope

The npm 404/permission failure is registry-side. `@kehto/paja` and `@kehto/cli` still need npm Trusted Publisher/package setup before rerunning the Release workflow can publish them.
