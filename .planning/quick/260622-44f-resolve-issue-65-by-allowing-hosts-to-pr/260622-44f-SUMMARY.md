# Quick Task 260622-44f: Resolve issue #65 by allowing hosts to provide read-only identity data to createIdentityService

## Summary

Issue #65 was valid and runtime-aligned: hosts should be able to provide read-only identity data without replacing the whole identity service, while Kehto remains out of the relay-query/indexing business.

Implemented optional `createIdentityService` provider hooks for `getProfile`, `getFollows`, `getList`, `getZaps`, `getMutes`, `getBlocked`, and `getBadges`. Existing empty/default fallbacks remain when hooks are absent. Provider failures return `.result` envelopes with `error` so the installed identity shim rejects instead of hanging.

Also fixed the Paja parity test to bind to the installed `@napplet/nap@0.13.0` dependency instead of an adjacent unreleased `napplet` checkout.
