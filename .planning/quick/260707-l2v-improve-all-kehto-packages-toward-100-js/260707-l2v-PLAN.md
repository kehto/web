# Quick Task 260707-l2v: Improve JSR Package Scores

## Goal

Move every published `@kehto/*` package as close as practical to a 100% JSR
score from repo-owned changes, and document remaining operator-owned JSR package
settings gaps.

## Evidence Baseline

Live JSR score pages on 2026-07-07 show:

- `@kehto/acl`: 76% — missing all-entrypoint module docs plus JSR settings/provenance.
- `@kehto/cli`: 64% — weak symbol docs plus JSR settings/provenance.
- `@kehto/firewall`: 82% — repo source score complete; only JSR settings/provenance missing.
- `@kehto/nip`: 41% — missing all-entrypoint module docs, one symbol-doc point,
  slow types, plus JSR settings/provenance.
- `@kehto/paja`: 47% — missing all-entrypoint module docs, symbol docs, plus
  JSR settings/provenance; dry-run also shows one slow type in current source.
- `@kehto/runtime`: 82% — repo source score complete; only JSR settings/provenance missing.
- `@kehto/services`: 76% — missing all-entrypoint module docs plus JSR settings/provenance.
- `@kehto/shell`: 52% — slow types plus JSR settings/provenance.
- `@kehto/wm`: 76% — weak symbol docs plus JSR settings/runtime compatibility.

## Tasks

1. Fix slow-type diagnostics without changing public behavior.
   - `packages/nip/src/5d/artifact-cache.ts`: annotate defaulted parameters.
   - `packages/paja/src/parity.ts`: annotate exported/public constant shape.
   - `packages/shell/src/*`: annotate exported singleton object types and any
     public constants reported by `jsr publish --dry-run`.

2. Improve JSR documentation scoring.
   - Add `@module` docs to every exported JSR entrypoint currently missing it.
   - Add concise JSDoc to public exported symbols in low-coverage packages
     (`cli`, `paja`, `wm`, and any nearby changed exports).
   - Keep docs useful and terse; do not add filler comments.

3. Validate package publish surfaces.
   - Run `pnpm dlx jsr publish --dry-run --allow-dirty` for every package.
   - Run package or repo type checks impacted by annotations.
   - Run `pnpm docs:check`, `pnpm test:unit`, `git diff --check`, and the
     pinned AI-slop changed-file scan.

4. Ship and summarize.
   - Add changesets for packages whose published source/docs changed.
   - Update this quick task with final evidence.
   - Commit, push, and open a PR.
   - Completion summary must include operator guide for JSR description,
     runtime compatibility, repository linkage/OIDC provenance, and any package
     versions that still need the release workflow.

## Final Status

Source-owned score gaps were addressed for `@kehto/acl`, `@kehto/cli`,
`@kehto/nip`, `@kehto/paja`, `@kehto/services`, `@kehto/shell`, and
`@kehto/wm`. `@kehto/firewall` and `@kehto/runtime` already had the repo-owned
JSR source factors complete in the live baseline, so they were left untouched.

Remaining gaps are operator/registry owned: JSR package description/runtime
compatibility/repository settings, OIDC provenance after publish, and the live
registry lag where `@kehto/cli` is still `0.2.11` and `@kehto/paja` is still
`0.6.3` even though the repo has `0.2.12` and `0.6.4`.
