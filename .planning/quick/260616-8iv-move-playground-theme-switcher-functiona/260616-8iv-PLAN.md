<objective>
Move the playground theme-switcher's FULL functionality from the sandboxed napplet into the HOST, mounted on the theme service node, and delete the napplet. This eliminates a sandbox-boundary violation: today the napplet sends a raw, un-ACL'd `window.parent.postMessage({ type: 'theme.set' /* 'n' in source */ })` that a bespoke `window.addEventListener('message')` handler in `apps/playground/src/main.ts` catches. That wire format is private and non-portable — theme APPLICATION is a host responsibility, and NAP-THEME is read-only for napplets.

Purpose: restore napplet/runtime cross-compatibility by routing theme switching through the existing host theme path (`relay.publishTheme` / `main-preferences.handleThemeMessage` -> `theme.changed`) with NO new wire verbs.

Output: a host-side discovery module, a host-side theme-switcher UI on the theme service card, removal of the raw seam, deletion of the napplet + its registration, and a fully-updated test/doc regression surface.
</objective>
