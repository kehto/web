# Quick Task 260623-a4i: Optimize Playwright CI Triggers and Relevant Spec Selection

## Context

The CI workflow currently runs the full Playwright suite on every `main` push and every pull request after build and unit tests pass. This is wasteful for docs/planning/package-only changes that cannot affect browser behavior, and it also misses an opportunity to run only directly relevant specs when the changed path is narrow.

## Plan

1. Add a deterministic changed-file selector script.
   - Return `run_e2e=false` for docs/planning/changeset-only changes.
   - Return specific spec paths for direct spec changes and narrow package/domain changes.
   - Return the full `tests/e2e` suite for shared runtime/shell/workflow/config changes where a narrow subset would be unsafe.

2. Wire CI to the selector.
   - Add a cheap detector job that runs on every CI event.
   - Make the Playwright job depend on the detector and skip unless `run_e2e=true`.
   - Pass selected specs to `pnpm test:e2e -- ...` when the selector narrows the suite.

3. Verify behavior.
   - Unit-test the selector rules.
   - Validate workflow syntax with `actionlint`.
   - Run build/type/unit/docs/slop/diff gates.
