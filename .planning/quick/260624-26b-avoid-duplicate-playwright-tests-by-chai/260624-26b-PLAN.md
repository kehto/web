# Quick Task 260624-26b: Avoid duplicate Playwright release runs

## Goal

Stop the normal release/publish path from running Playwright after CI already ran it for the same `main` commit.

## Plan

1. Change `.github/workflows/publish.yml` from a direct `push` trigger on `main` to a `workflow_run` trigger that listens for successful `CI` completions on `main`.
2. Remove duplicate publish-workflow validation steps: build, type-check, unit tests, docs check, Playwright browser setup, and Playwright execution. `pnpm publish-packages` still builds package artifacts during the actual publish path.
3. Add a job-level guard so publish work only runs when the upstream CI conclusion is `success`.
4. Remove duplicate unit/Playwright gates from fallback `.github/workflows/release.yml` while preserving build/type-check and publish/repair controls.
5. Validate workflow syntax locally without triggering a release.

## Explicit Non-Goal

Do not manually dispatch, rerun, tag, or otherwise trigger a release. This branch should only open a PR; the changed workflow behavior will be debugged on the next real release.
