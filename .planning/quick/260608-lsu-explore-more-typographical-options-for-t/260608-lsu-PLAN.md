# Quick Task 260608-lsu: Kehto Typography Exploration

## Direction

Explore typography that makes Kehto feel like a contained, protective home for experimental apps without losing the modern runtime/tooling signal.

Chosen direction: **humanist cradle, technical shell**.

- Brand wordmark: move away from the oversized old-style serif that read awkwardly in screenshots. Use a restrained humanist sans stack that can feel warm on platforms with Optima/Candara and stays clean on Linux through FreeSans.
- Body text: use a modern interface sans stack for readability and product clarity.
- Runtime/path labels: keep the technical layer in a mono stack so the page still signals developer/runtime infrastructure.

## Tasks

1. Preserve the in-progress contour interaction refinements from the prior feedback.
   - Add stronger settled mouse influence after the entrance completes.
   - Add passive transparency pulse bands.
   - Add active top-to-bottom pulses on playground/docs hover and focus.

2. Apply the typography direction to the static landing page.
   - Introduce brand/copy/mono font variables.
   - Replace the serif wordmark/card headings with the humanist brand stack.
   - Move body and detail labels onto the copy/mono stacks.
   - Keep responsive text fixed by breakpoint rather than viewport-scaled typography.

3. Verify.
   - Run focused static guard tests and syntax checks.
   - Run site build/audit and broader type/unit/lint checks as time permits.
   - Use live browser screenshots and canvas checks for desktop/mobile, settled pointer wake, and hover pulse behavior.
