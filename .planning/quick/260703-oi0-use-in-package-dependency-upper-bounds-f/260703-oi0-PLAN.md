# Quick Task 260703-oi0: Inclusive Dependency Upper Bounds

## Issue

kehto/web#139 points at `packages/services/package.json` peer/dev dependency
ranges such as `>=0.23.0 <0.26.0` and asks that dependency ranges use `<=` to
make the maximum supported version clear.

## Scope

- Update every public package manifest that uses `>=... <...` dependency ranges.
- Use the current effective support windows:
  - `@napplet/core` / `@napplet/nap`: `>=0.23.0 <=0.26.x`
  - `nostr-tools`: `>=2.23.3 <=2.x`
- Sync package docs and lockfile importer specifiers.
- Add changesets for packages whose published manifest metadata changes.
