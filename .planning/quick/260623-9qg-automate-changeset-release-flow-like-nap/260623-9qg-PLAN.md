# Quick Task 260623-9qg: Automate Changeset Release Flow Like Napplet/Web

## Context

`napplet/web` uses `changesets/action@v1` on `push` to `main` to create or update a Version Packages PR when pending changesets exist, then publish when that release PR is merged. Kehto currently has Changesets and a tag-triggered `release.yml`, but no push-to-main Changesets action workflow.

## Plan

1. Add a `Publish` workflow on `push` to `main`.
   - Use `changesets/action@v1` with `version: pnpm version-packages`.
   - Use Kehto's existing `pnpm publish-packages` for npm publishing.
   - Use OIDC permissions for npm trusted publishing and JSR publishing.

2. Preserve Kehto registry behavior.
   - Keep existing tag/manual `release.yml` as a fallback path.
   - Publish JSR only after the Changesets action reports npm packages were published.
   - Reuse `scripts/sync-jsr-versions.mjs` and topological serial JSR publish command.

3. Verify and ship to current PR branch.
   - Validate workflow action versions.
   - Run local syntax/grep checks and relevant gates.
   - Commit, push, and check PR CI.
