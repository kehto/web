# Summary

Fixed the GitHub Pages workflow ordering after the post-release publish run failed while packing the Pages artifact.

## Root Cause

`playground-pages.yml` rebuilt `@kehto/playground` with `PLAYGROUND_BASE_PATH=/web/playground/`, then ran `pnpm type-check`. The root `type-check` script starts with `pnpm build`, which can overwrite `apps/playground/dist` before `pnpm build:pages` asserts the Pages base.

## Fix

Moved the Pages-base playground rebuild after `pnpm type-check` and before docs/build packing.

## Verification

- `PLAYGROUND_BASE_PATH=/web/playground/ VITEPRESS_BASE=/web/docs/ pnpm type-check`
- `PLAYGROUND_BASE_PATH=/web/playground/ pnpm --filter @kehto/playground build`
- `PLAYGROUND_BASE_PATH=/web/playground/ VITEPRESS_BASE=/web/docs/ pnpm docs:check`
- `PLAYGROUND_BASE_PATH=/web/playground/ VITEPRESS_BASE=/web/docs/ pnpm build:pages`
- `pnpm audit:pages`
