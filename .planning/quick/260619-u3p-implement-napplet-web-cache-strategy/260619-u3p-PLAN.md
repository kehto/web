# Implement Napplet Web Cache Strategy - Plan

## Goal

Implement the Cache Storage based napplet artifact cache from
`docs/concepts/napplet-web-cache-strategy.md` without changing NIP wire
contracts or weakening verification.

## Tasks

1. Add a browser-cache adapter in `packages/nip/src/5d/artifact-cache.ts`.
   - Versioned cache namespace, synthetic same-origin request keys, and JSON
     index response stored in the same Cache Storage namespace.
   - Immutable blob records keyed by SHA-256.
   - Immutable aggregate records keyed by `(dTag, aggregateHash)`.
   - Mutable coordinate records keyed by `(kind, pubkey, dTag)` with foreground
     and background TTL helpers.
   - In-memory active aggregate tracking so currently running sessions are not
     pruned.

2. Extend `resolveNapplet()` with an optional cache.
   - Verify manifest signature and aggregate before any cache write.
   - On blob cache hits, re-run `verifyBlobHash()` before accepting bytes.
   - On cache miss, fetch from the caller and verify exactly as today.
   - Write blobs and aggregate only after all verification succeeds.
   - Surface cache write failures through diagnostics and continue uncached.

3. Wire the playground resolver to opt into the cache only when available.
   - Feature-detect `globalThis.caches`.
   - Use `navigator.storage.estimate()` when available for quota sizing.
   - Fall back to network-only resolution if Cache Storage is missing or the
     cache adapter cannot open.

4. Add focused tests.
   - Unit tests for cache hit, cache miss, verification failure, coordinate
     freshness, quota pruning/LRU/refcount, and network-only fallback.
   - Playground resolver test proving the cache adapter is optional and does not
     change the trust path.

5. Update public package docs only if exported behavior needs clarification.
   - No strategy-doc rewrite unless implementation materially clarifies the
     already accepted strategy.
   - Add a changeset for `@kehto/nip` and `@kehto/playground` if shipped output
     changes.

6. Verify and ship.
   - `pnpm build`
   - `pnpm type-check`
   - `pnpm test:unit`
   - `pnpm docs:check` if docs changed
   - `git diff --check`
   - AI-slop gate (`npx aislop scan`)
   - Commit with Lore trailers, push `feat/napplet-web-cache`, open a PR.

## Non-Goals

- No localStorage usage.
- No IndexedDB artifact cache.
- No service worker requirement.
- No runtime/shell protocol or NAP message changes.
- No stale-while-revalidate launch path in the first implementation unless the
  existing host policy explicitly opts in.
