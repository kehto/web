# Plan 98-01: NAP-SERIAL Runtime Parity

## Goal

Wire `serial.*` through Kehto's normal service-backed NAP path with shell-owned sessions, deterministic Paja/playground behavior, and focused proof.

## Implementation Slices

1. Runtime dispatch
   - Register `serial` in `createNapEnvelopeDispatcher`.
   - Add `serial` to service-only domain handlers.
   - Add focused runtime tests proving `serial.open`, `serial.write`, and `serial.close` route to the service and missing service does not throw.

2. Shell capability advertisement
   - Add `SerialHooks` with `isAvailable()`.
   - Advertise `serial` in `domains` and `naps` only when the hook is available.
   - Extend shell-init tests for available/disabled behavior.

3. Reference service
   - Add `createSerialService` to `@kehto/services`.
   - Host hooks: `open`, `write`, `close`; optional event sender wiring for pushed events if needed by deterministic demos.
   - Missing hooks return result envelopes with `error`, not thrown exceptions.
   - Unit tests cover success, unsupported hooks, hook exceptions, and unknown messages.

4. Paja support
   - Add `serial` to Paja parity and simulation domains.
   - Wire a deterministic in-memory serial service that opens a synthetic session, records writes, and closes sessions.
   - Keep UI unchanged.

5. Playground demo
   - Add `serial-demo` napplet using raw serial envelopes until an in-repo helper consumer exists.
   - Register it in demo definitions, gateway/static guards, and e2e inventory checks.
   - Add Playwright test proving open/write/close results appear through the real shell.

6. Docs/release
   - Update package docs/READMEs and NIP-5D raw-envelope policy.
   - Add a changeset for packages with shipped output changes.

## Verification

- Focused TypeScript for touched packages/apps.
- Focused Vitest for runtime/shell/services/static guards.
- Focused Playwright for `serial-demo`.
- Full `pnpm build`, `pnpm type-check`, `pnpm test:unit`, `pnpm docs:check`.
- Full CI-shaped `CI=1 pnpm test:e2e`.
- `npx --no-install aislop scan --changes --base origin/main`.
- `git diff --check`.

## Stop Condition

Phase 98 is complete when branch `feat/nap-serial-parity` is pushed, a stacked PR is open on `feat/nap-lists-parity`, local gates are green, and remaining missing 0.20.0 domains are explicitly limited to BLE and WebRTC.
