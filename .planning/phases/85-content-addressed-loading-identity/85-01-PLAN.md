# Phase 85 — Plan 85-01: Content-Addressed Loading & Identity

**Milestone:** v1.20 NIP-5D Content-Addressed Runtime Resolution
**PR:** PR2 (branch `feat/nip5d-runtime-srcdoc`, stacked on `feat/nip5d-resolver`).
**Requirements:** IDENTITY-01..03, LOAD-01..07, DOCS-01..02

## Goal

Load napplets end-to-end via relays → Blossom → verify → `iframe.srcdoc`, with no
gateway in the trust path; identity is the computed `(dTag, aggregateHash)` from
verified bytes; every failure rejects. CSP holds under srcdoc; storage/relay/ACL/
firewall mediation unchanged; docs repinned to branch-HEAD NIP-5D.

## Tasks

### Task 1 — Build migration (`shared-vite-config.ts`) (LOAD-06)
- Emit NIP-5A `path` tags (`["path","/index.html","<sha256>"]`) + ONE
  `["x","<hex>","aggregate"]` tag (over real files only). Kind `35128`→`35129`.
- Drop synthetic `config:schema` / `connect:origins` from the aggregate (NIP-5A =
  path tags only) and drop the `napplet-aggregate-hash` `<meta>` injection so the
  served `/index.html` blob hashes to its `path` tag (no chicken-and-egg).
- Keep `config`/`connect` as declarative tags; keep signed-event output.

### Task 2 — In-repo relay + Blossom sim (`vite.config.ts`) (LOAD-01)
- `GET /napplet-relay/:author/relay-list` → signed kind-10002 NIP-65 event whose
  write relay points at the sim relay base.
- `GET /napplet-relay/:dTag` → the signed kind-35129 manifest event
  (`.nip5a-manifest.json`).
- `GET /blossom/:sha256` → the matching file bytes (404 on unknown hash).
- Gateway HTML/CSP-header routes retired from the trust path (kept only as a
  legacy accelerator if needed; never trusted).

### Task 3 — `@kehto/nip/5d` browser fetch adapters + CSP inject helper (pure, TDD)
- A small playground module: `fetchRelayList`, `selectManifestRelays`
  (via `@kehto/nip/65`), `fetchManifestEvent`, `fetchBlobBytes` (Blossom),
  `injectCspMeta(html, origins)` → inserts `<meta http-equiv="Content-Security-Policy">`.
- Unit-test `injectCspMeta` (connect-src from origins; `'none'` when empty) and
  the resolve wiring against fakes (sig/blob/aggregate/missing-index rejection,
  srcdoc text, computed identity mapping).

### Task 4 — `shell-host.ts` srcdoc loader (IDENTITY-01..03, LOAD-02..04)
- Replace `fetchGatewayMetadata`+`iframe.src` with: NIP-65 select → fetch manifest
  event → `resolveNapplet({event, fetchBlob})` → assert `event.pubkey===author` &
  `dTag` → inject CSP meta → `iframe.srcdoc`. `sandbox="allow-scripts"`, never
  `allow-same-origin`.
- Identity `(dTag, aggregateHash)` from the resolved bytes feeds
  `registerSessionEntry` / `markSourceDerivedIdentity`. Reject (throw) on any
  resolution failure — no iframe shown with unverified bytes.
- Retire `GatewayNappletMetadata` / `htmlUrl`.

### Task 5 — `main.ts` adapt (LOAD-05)
- `pregrantBeforeGatewayNavigation` → pre-resolve connect grant in `connectStore`
  (so injected CSP meta includes it); drop `GatewayNappletMetadata` type usage.

### Task 6 — Update tests (LOAD-07)
- `tests/unit/playground-gateway-guard.test.ts` → assert srcdoc model.
- `tests/e2e/gateway-artifact-parity.spec.ts`, `nip5d-contract-conformance.spec.ts`,
  `profile-open.spec.ts` → srcdoc + relay/Blossom resolution.

### Task 7 — Docs/spec repin (DOCS-01..02)
- `RUNTIME-SPEC.md` + `specs/NIP-5D.md`: repin branch-HEAD NIP-5D, kinds
  `35129`/`15129`/`5129`, relays→Blossom→verify→srcdoc, computed identity. No
  "backwards compatibility" language.
- `@kehto/runtime` / `@kehto/shell` READMEs/CHANGELOGs where changed; changesets.
