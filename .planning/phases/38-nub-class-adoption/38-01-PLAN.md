<objective>
Establish the type-level and session-level foundation for synchronous class posture. Breaking-expand the `onNip5dIframeCreate` host-app hook return type with a `class: NappletClass` field (CLASS-01), thread that class value into `SessionEntry` in both the runtime and shell session registries (CLASS-02), and emit the resolved class inline inside `shell.init` (CLASS-02 — no async `class.assigned` envelope per C-01 prevention). Update the demo `shell-host.ts` registerSessionEntry() call sites to stamp `class: null` (permissive default, per D2) so no existing E2E spec regresses. Produce a minor-bump changeset documenting the breaking hook change for downstream coordination (hyprgate in parallel per user decision).

Purpose: Phase 38's entire enforcement gate (Plan 38-02) reads `class` from the session entry. Without this plan landing the type surface and session propagation, the gate has nothing to read. Without `shell.init` carrying the class inline, the shim would receive class asynchronously — creating the C-01 pre-assignment race where NUB requests arriving between `shell.ready` and a later `class.assigned` bypass class enforcement. This plan is also the scope of the breaking API change; Plan 38-02 does NOT expand the public hook.
