## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host wiring/grant callback → shell environment builder | Host policy narrows live capabilities for one trusted creation identity. |
| Shell environment builder → iframe init | Authoritative domains/services cross into untrusted frame memory. |
| Frame A ↔ Frame B | Independent source registrations must never share mutable environment state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-101-08 | Elevation | `CapabilityHooks.resolveEnvironment` | high | mitigate | Intersect grants against live/undisabled availability; reject additions, aliases, case changes, duplicates, and prefix-adjacent values by exact membership. |
| T-101-09 | Information Disclosure / Tampering | `resolveShellEnvironment` | high | mitigate | Copy and freeze fresh domains/services per call and identity; two-frame tests mutate every exposed/input array and assert no cross-frame effect or shared reference. |
| T-101-10 | Spoofing | `handleShellReady` identity handoff | high | mitigate | Resolve creation-time identity once from the registered source and pass it explicitly to both session and environment construction. |
| T-101-11 | Elevation | identity-less registration | high | mitigate | Inherit Plan 101-01 fail-closed behavior: no identity means no resolver call, session, or init. |
| T-101-12 | Tampering | duplicate ready | high | mitigate | Return before environment reconstruction for the same registration id; only host re-registration starts a new snapshot lifecycle. |
</threat_model>
