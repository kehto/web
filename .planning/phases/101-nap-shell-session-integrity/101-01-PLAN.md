## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iframe `postMessage` → ShellBridge | Untrusted frame data and source enter the trusted host. |
| ShellBridge → runtime session registry | First ready causes the privileged session-establishment side effect. |
| Runtime ingress → ACL/firewall/services/domains | Capability traffic crosses into policy and behavior only after session establishment. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-101-01 | Spoofing / Elevation | `handleShellReady` | high | mitigate | Ignore ready payload claims and bind only the registered `MessageEvent.source` plus creation-time dTag/aggregateHash; prove forged extra fields cannot re-key the session. |
| T-101-02 | Elevation | `createMessageHandler` | high | mitigate | Return before ACL, firewall, service, or domain work when `sessionRegistry.getEntryByWindowId` is absent; assert every spy remains untouched. |
| T-101-03 | Tampering / Elevation | `initSent` and SessionRegistry | high | mitigate | Keep registration-id-scoped ready idempotency and assert duplicate ready neither overwrites the session nor posts init again. |
| T-101-04 | Information Disclosure | per-frame init environment | high | mitigate | Plan 101-02 creates immutable source-specific snapshots and two-frame isolation tests before host expansion. |
| T-101-05 | Spoofing | injected init receiver | high | mitigate | Plan 101-03 retains parent-source-only, first-init-only acceptance and adds forged sibling/child init vectors. |
| T-101-06 | Elevation | identity-less registered frame | high | mitigate | Withhold both session and init when creation-time identity is unavailable; assert the frame remains inert. |
| T-101-07 | Spoofing / Tampering | originRegistry registration boundary | high | mitigate | Resolve source and registration identity from the host registry on every message and prove ready payloads cannot reassign them; allow only explicit host re-registration to start a new lifecycle. |
