## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| verified napplet bytes → Paja bootstrap | Untrusted napplet code executes beside host-injected receivers without becoming the source of host identity or grants. |
| iframe contentWindow → Paja host registry | An untrusted browsing context must be bound to its trusted creation identity before handshake traffic is accepted. |
| Paja host wiring → shell.init | Mutable host configuration is converted into an immutable per-frame environment snapshot. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-101-04-01 | Spoofing | Paja iframe registration | high | mitigate | Register contentWindow with trusted creation identity before srcdoc execution and reject identity-less ready. |
| T-101-04-02 | Elevation of Privilege | Paja use of `resolveShellEnvironment` | high | mitigate | Call the shared host resolver with trusted OriginIdentity, intersect live host wiring with disables and manifest requirements, and never derive grants from manifest or ACL operation rules. |
| T-101-04-03 | Information Disclosure | Paja environment snapshots | high | mitigate | Build immutable isolated snapshots per frame identity and test that disabled/unwired membership cannot leak across frames. |
| T-101-04-04 | Tampering | Paja reload registration | high | mitigate | Preserve registration-id replacement so stale sessions cannot authorize the reloaded document, while same-registration ready remains idempotent. |
| T-101-04-05 | Tampering | verified srcdoc provenance | high | mitigate | Keep injected bootstrap outside verified artifact bytes and retain the existing static/e2e provenance assertion. |
</threat_model>
