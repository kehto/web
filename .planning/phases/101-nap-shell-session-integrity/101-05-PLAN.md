## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gateway output → verified loader | Gateway output is untrusted acceleration data and cannot establish napplet identity or grants. |
| verified srcdoc → injected playground bootstrap | Host receivers execute beside verified bytes but remain outside the identity-bearing artifact. |
| iframe contentWindow → playground shell host | Each untrusted frame must be registered with immutable creation identity before handshake or domain traffic. |
| mutable demo controls → per-frame environment | Host UI/service toggles are copied into isolated environment snapshots rather than shared between frames. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-101-05-01 | Spoofing | playground source registration | high | mitigate | Bind contentWindow to trusted creation identity before ready and reject identity-less or reassigned sources. |
| T-101-05-02 | Information Disclosure | concurrent playground environments | high | mitigate | Freeze isolated domain/service snapshots and exercise different simultaneous frame profiles in Playwright. |
| T-101-05-03 | Elevation of Privilege | playground use of `resolveShellEnvironment` | high | mitigate | Call the shared host resolver with trusted OriginIdentity, resolve membership from concrete live wiring, apply disables before the host subset callback, and test all receiver/support/init boundaries. |
| T-101-05-04 | Tampering | gateway and srcdoc provenance | high | mitigate | Retain static and browser assertions that verified bytes establish identity while gateway data and injected bootstrap cannot. |
| T-101-05-05 | Denial of Service | duplicate ready/reload | medium | mitigate | Prove duplicate ready is idempotent and a new registered reload receives exactly one init. |
</threat_model>
