## DOM Contract (Authoritative ‚Äî from CONTEXT.md Area 3)

This is the primary contract for this phase. All sentinel IDs and state strings are locked.

### Status Sentinel

| Element ID | Type | State Sequence |
|------------|------|----------------|
| `#media-controller-status` | text content | `connecting...` ‚Üí `authenticated` ‚Üí `session-ready` ‚Üí `playing` or `paused` |

State transitions are one-directional during the session lifecycle. `playing` and `paused` are toggled by button interaction or incoming `media.command` from the shell.

### Interactive Buttons

| Element ID | Type | Trigger | Effect |
|------------|------|---------|--------|
| `#media-controller-play` | `<button>` | User click | Calls `mediaReportState({ status: 'playing' })`; status sentinel transitions to `playing` |
| `#media-controller-pause` | `<button>` | User click | Calls `mediaReportState({ status: 'paused' })`; status sentinel transitions to `paused` |

### Counter and Command Sentinels

| Element ID | Type | Description |
|------------|------|-------------|
| `#media-controller-command-count` | integer text | Increments on each shell-to-napplet `media.command` delivery (evidence of reverse command path) |
| `#media-controller-last-command` | text content | Most-recent command string received from shell (e.g. `"play"`, `"pause"`) |

### Audio Element

| Element | Attributes | Purpose |
|---------|-----------|---------|
| `<audio>` (silent loop) | Inside napplet iframe; `autoplay=false`; played programmatically when session becomes active | Napplet's own playable audio surface; exercises actual audio pipeline end-to-end |

### Hard-Coded Demo Metadata

```ts
{ title: 'Kehto Demo Track', artist: 'v1.4 Media', mediaType: 'audio' }
```

No artwork field. Artwork is an optional `MediaMetadata` extension surface; deferred to Phase 28 DOCS-05.

## E2E Interaction Contract

The following interaction sequence is what `tests/e2e/media-controller.spec.ts` (E2E-13) validates. Included here so the executor has a complete picture of required DOM behavior.

1. Page loads ‚Üí `#media-controller-status` contains `connecting...`
2. AUTH completes ‚Üí `#media-controller-status` contains `authenticated`
3. Session create completes ‚Üí `#media-controller-status` contains `session-ready`
4. Click `#media-controller-play` ‚Üí `#media-controller-status` contains `playing`; `navigator.mediaSession.playbackState === 'playing'`
5. Click `#media-controller-pause` ‚Üí `#media-controller-status` contains `paused`; `navigator.mediaSession.playbackState === 'paused'`
6. `navigator.mediaSession.metadata.title === 'Kehto Demo Track'` asserted after any state transition
7. Each shell-side `media.command` delivery ‚Üí `#media-controller-command-count` increments; `#media-controller-last-command` reflects command string

Capability-gate hook: `window.__grantMediaControl__(windowId?) ‚Üí boolean` must be installed in `bootShell()` (parallel to `window.__grantKeysForward__`). Spec calls this before asserting `session-ready`.
