**Section 4: SessionEntry Identity Anchor (RT-04)**

4.1 **Background** — Under RUNTIME-SPEC v2.0.0, `SessionEntry.pubkey` was set to the AUTH-derived ephemeral keypair public key after successful handshake. This pubkey served three roles:
1. Session authentication token (proved napplet completed AUTH)
2. ACL lookup key component (first segment of `pubkey:dTag:hash`)
3. Unique session identifier (different instances of same napplet got different pubkeys)

NIP-5D eliminates the AUTH keypair. SessionEntry.pubkey has no natural value. This section documents the design decision for what replaces it.

4.2 **Current SessionEntry Schema** — Show the exact TypeScript type from session-registry.ts with line numbers. Document each field's purpose and whether it survives migration:
- pubkey: AUTH keypair public key -> NEEDS REPLACEMENT
- dTag: napplet app identifier -> UNCHANGED
- aggregateHash: version hash -> UNCHANGED
- windowId: iframe window reference -> UNCHANGED (gains importance)
- authenticated: boolean flag -> REMOVE (no AUTH to complete)
- Other fields from the actual source

4.3 **Design Options** — Present the three candidate replacements for SessionEntry.pubkey:

Option A: **windowId as pubkey** — Set pubkey to the windowId string. This is the simplest change and preserves the "unique per session" property. The ACL composite key would be `windowId:dTag:hash`. Downside: windowId is runtime-ephemeral (changes every page load), so persisted ACL entries would not survive across sessions.

Option B: **Empty string** — Set pubkey to '' (empty). ACL lookup uses `dTag:hash` (per ACL-MIGRATION.md). This is the cleanest break — pubkey is acknowledged as meaningless under NIP-5D. Downside: code that checks `if (session.pubkey)` would evaluate to false, which may be the correct signal ("not authenticated") but could break existing guards.

Option C: **Remove pubkey field entirely** — Delete SessionEntry.pubkey. This is the most aggressive option and forces all consumers to update. Pair with Identity.pubkey being optional (per ACL-MIGRATION.md). Downside: largest code surface change; incompatible with legacy mode where AUTH napplets still set pubkey.

4.4 **Chosen Design: Option B (empty string) with deprecation** — Document the rationale:
- Aligns with ACL-MIGRATION.md decision: Identity.pubkey is optional, toKey() ignores it, composite key is `dTag:hash`
- Preserves backward compatibility: legacy AUTH napplets can still set pubkey to the derived key
- Allows gradual removal: pubkey field can be removed in a future cleanup pass once legacy support is dropped
- SessionEntry gains `identitySource: 'auth' | 'source'` discriminant field to distinguish NIP-5D sessions from legacy AUTH sessions
- `authenticated` boolean replaced by `identitySource` check: `session.identitySource === 'auth'` means legacy-authenticated; `session.identitySource === 'source'` means NIP-5D identity-at-creation

4.5 **New SessionEntry Schema** — Show the target TypeScript type:
```typescript
interface SessionEntry {
  /** @deprecated Use dTag + hash for identity. Empty string for NIP-5D sessions. */
  pubkey: string;
  dTag: string;
  aggregateHash: string;
  windowId: string;
  /** How identity was established: 'source' (NIP-5D) or 'auth' (legacy handshake) */
  identitySource: 'auth' | 'source';
  // ... other fields unchanged
}
```

4.6 **Session Creation Flow (NIP-5D)** — Document the new session creation path:
- Shell creates iframe from NIP-5A manifest
- Shell calls originRegistry.register(iframe.contentWindow, windowId, { dTag, aggregateHash })
- Shell (or runtime) creates SessionEntry with: pubkey='', dTag, aggregateHash, windowId, identitySource='source'
- First message from napplet is handled immediately (no AUTH queue)
- Contrast with legacy flow: REGISTER -> IDENTITY -> AUTH -> SessionEntry with pubkey=derivedKey, identitySource='auth'

4.7 **Impact on Downstream Packages** — Brief notes on how this decision affects:
- @kehto/acl: Already aligned — Identity.pubkey optional, toKey() uses dTag:hash
- @kehto/shell: SessionEntry changes in session-registry require updated shell-bridge session creation
- @kehto/services: Services that check session.pubkey must handle empty string / use identitySource discriminant
