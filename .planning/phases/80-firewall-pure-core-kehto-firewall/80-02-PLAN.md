<objective>
Implement the firewall's built-in defaults (`defaults.ts`) and the pure config mutation + serialization layer (`config.ts`), each with co-located vitest tests. This is the immutable-state half of the engine, mirroring acl's `capabilities.ts` (defaults) and `mutations.ts` (mutations + serialize/deserialize). It depends only on the types defined in Plan 01.

Purpose: The engine in Plan 03 calls `defaultConfig()` and `createState()` from here and applies the default rate/burst limits and unfocused multiplier. The runtime in Phase 81 persists config via `serialize`/`deserialize` — those must round-trip losslessly and defend against poisoned config strings (the only untrusted input to the whole package).

Output: `defaults.ts` (conservative built-in limits, `flag` default exceed-action, `block` default burst action, unfocused multiplier, `defaultConfig()`, `createState()`), `config.ts` (immutable `setPolicy`/`setRateLimit`/`addMatcher`, `serialize`, defensive `deserialize`), and tests proving CORE-03, CORE-04, BURST-02 default, and the RATE-03 global-fallback can be configured.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted config string → `deserialize()` | The ONLY untrusted input to the entire @kehto/firewall package. A poisoned or malformed config string reaches `deserialize`. No network, no eval, no file I/O. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-01 | Tampering | `deserialize(json)` in config.ts | mitigate | Defensive parse: try/catch JSON.parse, shape-validate every top-level + nested field, rebuild a validated config, fall back to `defaultConfig()` on any failure. Never throw. Mirrors acl mutations.ts deserialize (lines 239-270). Verified by malformed-input and structurally-invalid tests. |
| T-80-DoS | Denial of Service | config mutation functions | accept | Mutations are O(size-of-config) shallow spreads over small in-memory structures; no unbounded growth, no recursion. Config is host-authored, not napplet-supplied. Low risk. |
| T-80-SC | Tampering | npm/pip/cargo installs | accept | No external packages installed (RESEARCH Package Legitimacy Audit: zero deps). No legitimacy checkpoint required. |
</threat_model>
