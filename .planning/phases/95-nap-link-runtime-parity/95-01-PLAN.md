# Plan 95-01: NAP-LINK Runtime Parity

## Goal

Add `NAP-LINK` support end to end without implementing any other missing NAP domain.

## Tasks

1. Add shell/runtime contracts.
   - Add `LinkHooks` / `LinkOpenerLike` to `@kehto/shell`.
   - Advertise `link` in `buildShellCapabilities()` only when `hooks.link?.isAvailable()` is true.
   - Add shell-init tests for advertised, unavailable, and disabled-domain cases.
   - Add `link` to runtime domain handlers and NAP dispatch.

2. Add `@kehto/services` reference implementation.
   - Create `packages/services/src/link-service.ts`.
   - Export `createLinkService` and option/result types.
   - Validate absolute URLs.
   - Allow only `http:` and `https:` by default.
   - Return `link.open.result` with `opened` or `denied`.
   - Add unit tests for allowed open, malformed URL, unsafe scheme, host-denied result, host exception, and unknown action.

3. Wire Paja.
   - Register `link` service in `createDevServices`.
   - Add `link` shell adapter hook when capability is enabled.
   - Add `link` to Paja parity metadata/service list.
   - Update parity tests.

4. Add playground napplet and host behavior.
   - Add `apps/playground/napplets/link-demo`.
   - Register it in `DEMO_NAPPLETS`.
   - Wire playground shell adapter/service for deterministic link behavior.
   - Add Playwright coverage for allowed and denied link results.

5. Release metadata and verification.
   - Add changesets for modified published packages.
   - Run focused unit tests first.
   - Run focused Playwright for `link-demo`.
   - Run full gates before PR: `pnpm build`, `pnpm type-check`, `pnpm test:unit`, `pnpm test:e2e`, docs check if docs changed, AI-slop scan, `git diff --check`.

## Stop Condition

`link` is proven with service/unit/static/Paja/playground/e2e evidence, no other missing NAP is implemented, and PR is open.
