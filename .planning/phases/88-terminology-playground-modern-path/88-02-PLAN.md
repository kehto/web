createShellSupports(capabilities) builds its set from capabilities.naps ONLY, normalizing each entry by stripping a leading `nap:` prefix. Therefore on a migrated napplet hosted by @kehto/shell (which advertises naps including 'inc' and 'inc:NAP-01..06'):
- supports('inc') === true        (bare domain in naps)
- supports('nap:inc') === true    (nap: prefix is stripped)
- supports('inc', 'NAP-01') === true   (protocol-qualified; naps carries 'inc:NAP-01')
- supports('inc:NAP-01') === true       (single-arg protocol form)
- supports('ifc') === false       (not in naps)
- supports('nub:inc') === false   (shim does NOT strip nub:; 'nub:inc' is not in naps)
- supports('nostrdb') === false
