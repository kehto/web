<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| napplet (sandboxed iframe) → runtime | Untrusted postMessage envelopes cross here; the firewall config + counters live runtime-side and must never be napplet-writable |
| runtime → host persistence | `firewallPersistence` is host-supplied; runtime treats load() output as untrusted (defensive deserialize) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-01 | Tampering | persisted FirewallConfig via firewallPersistence.load() | mitigate | `deserialize` is defensive (never throws, falls back to `defaultConfig()` — Phase 80 T-80-01). `load()` wraps in best-effort try/catch and resets to default on throw. |
| T-81-02 | Denial of Service | container counter growth | accept | O(1) fixed token-bucket state per (dTag, opClass) by engine design (RATE-02); counters are ephemeral and reset on reload. No unbounded growth. |
| T-81-SC | Tampering | npm/pnpm installs | mitigate | Only new dep is in-repo workspace `@kehto/firewall` (Phase 80, 87 green tests). No external registry package — Package Legitimacy Gate N/A per 81-RESEARCH.md. |
</threat_model>
