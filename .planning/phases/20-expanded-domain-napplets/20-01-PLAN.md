<objective>
Implement an in-memory mock relay pool for the demo (per CONTEXT D-USER-01) that satisfies NAP-06's `feed` napplet delivery contract without requiring real relay traffic. Replace the current no-op stub relay pool in `createDemoHooks()` with a pool that:

   Each event has:
   - `id`: 64-char hex string derived from the event index (e.g., `'1'.repeat(64)`, `'2'.repeat(64)`, ..., `'5'.repeat(64)`)
   - `kind`: 1
   - `created_at`: `Math.floor(Date.now() / 1000) - (5 - index)` so events are time-ordered
   - `tags`: `[['t', 'demo-feed']]`
   - `sig`: `'0'.repeat(128)` (fixture-only — demo does not verify signatures on received events)

2. **Published event store** — maintain a module-level `const publishedEvents: NostrEvent[] = []` array that records `relay.publish` calls. Expose this via an optional `getPublishedEvents()` method on the returned pool for future test inspection (not required by v1.3 specs but harmless).

3. **getRelayPool() return object** — implement:
   - `subscription(_relays, filters, options)`: coerce `filters` to `NostrFilter[]` (wrap single filter in array). For each fixture event, test it against each filter:
     - `kinds` filter (if present): event.kind must be in `filters.kinds`
     - `limit` (if present): cap the total matching events at `filter.limit`
     - Any other filter fields MAY be ignored in v1.3 (authors, since, until, '#t') — keep the matcher minimal.
     - For every matching event, invoke `options?.onevent?.(event)` synchronously via a `queueMicrotask(() => options.onevent(event))` (microtask, not sync-in-loop — the runtime's subscription dispatcher expects the onevent callback to be invoked AFTER subscribe returns).
     - After all matching events have been queued, `queueMicrotask(() => options?.oneose?.())` to emit EOSE.
     - Return `{ close: () => { /* no-op — microtasks already fired */ } }`.
   - `publish(_relays, event)`: push `event` onto `publishedEvents`; return `[Promise.resolve('ok')]` to match nostr-tools shape.
   - `request(_relays, _filters)`: no-op — return `{ subscribe: () => ({ unsubscribe: () => {} }) }` matching existing stub.

4. **trackSubscription / untrackSubscription / openScopedRelay / closeScopedRelay / publishToScopedRelay / selectRelayTier** — keep as no-ops matching the existing stub (return `undefined` / `false` / `[]` as appropriate). NAP-06 does not require scoped-relay wiring.

5. **Imports** — use `type { NostrEvent, NostrFilter }` from `@napplet/core` (already a transitive dep via @kehto/shell). Do NOT add `@napplet/core` to apps/demo/package.json as a direct dep (Pitfall 4 in research/PITFALLS.md — dedupe risk).
