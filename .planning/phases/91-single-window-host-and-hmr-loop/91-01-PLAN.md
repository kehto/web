<objective>
Turn the Phase 90 static host page into a real single-window development host: one sandboxed iframe, top bar, bottom bar, browser bootstrap script, production-shaped shell.ready/shell.init handshake, and reload/reinitialize controls that preserve the target app's own HMR. The implementation must keep Phase 92 service parity out of scope while making the iframe lifecycle and shell bootstrap authentic enough for a real Playwright fixture.
</objective>

<discovered_patterns>
- The e2e harness documents the safe iframe sequence: create iframe, append it to obtain `contentWindow`, register origin/session identity before setting `src`, then set `src` last so first napplet postMessages are routable.
- Production shell bootstrap uses `shell.ready` -> `shell.init` with `buildShellCapabilities(hooks)`, structured `domains`/`protocols`, legacy `naps`, `sandbox`, and service names.
- The dev runtime can preserve app HMR by leaving the iframe pointed at the target URL and reloading/reinitializing only the host runtime state around the still-running target process.
- Generated TypeDoc and package docs are already wired in Phase 90; new public browser-host helpers need exported types/JSDoc if exposed.
</discovered_patterns>
