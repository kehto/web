# Plan 72-01: Aislop Critical Warning Cleanup

**Phase:** 72 - Aislop Critical Warning Cleanup
**Status:** Ready
**Created:** 2026-05-24

## Goal

Reduce the local `aislop` scan from Critical status after the corrected scanner invocation, without broad behavior changes or aggressive dependency alignment.

## Cleanup Plan

1. Keep behavior locked with the existing unit suite before cleanup edits.
2. Run `npx --no-install aislop fix` to apply the scanner's normal safe auto-fixes for narrative/trivial comments and other fixable warnings.
3. Review the generated diff before manual edits.
4. Re-run `npx --no-install aislop scan -d` and classify remaining warnings.
5. Apply targeted manual cleanup only for local, low-risk leftovers:
   - unused imports/locals
   - duplicate imports
   - narrowable double casts
   - thin wrappers that do not preserve public or script-local clarity
6. Defer broad long-function, large-file, duplicate-block, and deep-nesting refactors unless a narrow behavior-locked extraction is obvious.
7. Verify with `pnpm type-check`, `pnpm build`, `pnpm test:unit`, final `npx --no-install aislop scan -d`, and `git diff --check`.

## Stop Condition

The phase stops when local `aislop` no longer reports Critical status, or when all remaining Critical contributors are non-fixable structural/dependency warnings that require an explicit policy threshold or future structural milestone.

## Commands

- `pnpm test:unit`
