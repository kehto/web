# Plan 99-01: NAP-BLE Runtime Parity

## Goal

Make `ble.*` work through Kehto's normal shell-owned service path and prove it in Paja and the playground.

## Requirements

BLE-01 through BLE-06.

## Steps

1. Add shell/runtime dispatch surface.
   - Add `BleHooks` to `@kehto/shell`.
   - Advertise `ble` in `domains` and `naps` only when `hooks.ble.isAvailable()` is true.
   - Register `ble` in runtime NAP dispatch and service-only domain routing.

2. Add `@kehto/services` reference handler.
   - Export `createBleService`.
   - Handle `open`, `services`, `read`, `write`, `subscribe`, `unsubscribe`, and `close`.
   - Return shaped upstream-compatible results for unavailable hooks, host exceptions, and unknown actions.

3. Wire deterministic host implementations.
   - Add in-memory BLE controller to Paja.
   - Move `ble` from deferred to advertised/required Paja parity metadata.
   - Add playground demo BLE service and capability hook.

4. Add playground `ble-demo`.
   - Build a small napplet that waits for `ble`, opens a session, lists services, reads bytes, writes bytes, subscribes, unsubscribes, closes, and renders stable DOM sentinels.
   - Add gateway/guard/README updates.

5. Verify.
   - Focused unit tests for service, shell advertisement, runtime dispatch, and static guards.
   - Focused Playwright test for `ble-demo`.
   - Full repo gates, changeset, commit, push, PR stacked on `feat/nap-serial-parity`.

## Acceptance

- `ble` absent when unwired and present when wired.
- `ble.*` routes to a registered service through runtime.
- Playground `ble-demo` shows successful open/services/read/write/subscribe/unsubscribe/close through the real shell.
- `webrtc` remains deferred.
