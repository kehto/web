<objective>
Emit the renamed `naps` handshake field (NAP vocabulary: bare `inc` +
`inc:NAP-01..06`) while dual-emitting the legacy `nubs` field for one back-compat
release (D2/D3/D4 advertised set / ALIGN-01..04), and prove the outcome with a
conformance test that feeds kehto-emitted capabilities into the REAL
`@napplet/shim@0.9.0` `createShellSupports` (D7/ALIGN-07). Stage the changeset
(D8/ALIGN-08).

Purpose: The 0.9.0 shim reads ONLY `capabilities.naps` and matches protocols via
`/^([^:]+):(NAP-\d+)$/`, adding the bare domain for any `domain:NAP-NN` entry.
Legacy (<=0.8.x) shims read `nubs`. Dual-emit lets napplets on either side of the
rename negotiate during the window. The conformance test is behavior verification,
not string matching.
Output: `naps`+`nubs` dual-emit, updated handshake tests, a shim-backed
conformance test, the shim added as a DEV dependency, and a changeset.
</objective>
