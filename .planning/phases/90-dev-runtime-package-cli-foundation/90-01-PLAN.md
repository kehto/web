# Phase 90 Plan 01: Dev Runtime Package and CLI Foundation

**Phase:** 90 — Dev Runtime Package and CLI Foundation  
**Milestone:** v1.22 Single-Window Development Runtime  
**Status:** Planned  
**Created:** 2026-06-21

## Goal

Create the first real `@kehto/dev-runtime` package slice: a publishable ESM workspace package with a typed option model, CLI entry, target URL / child command handling, generated host config, and tests. This phase deliberately does not try to complete the UI or full NAP wiring; it creates the stable package/CLI foundation those later phases will use.

## Requirements Covered

- **DEVRT-01**: publishable `@kehto/dev-runtime` package exists.
- **DEVRT-02**: CLI supports package-manager `dev` usage through target URL or child-command mode.
- **DEVRT-03**: framework-agnostic HMR strategy is encoded as target URL loading, not framework ownership.
- **DEVRT-04**: typed programmatic helpers expose the same option contract.
- **VERIFY-01**: unit coverage for option parsing, config validation, URL/command handling, and host config generation begins here.

## Implementation Scope

1. Add `packages/dev-runtime/` to the existing package pattern with:
   - `package.json`
   - `tsconfig.json`
   - `tsup.config.ts`
   - `src/index.ts`
   - `src/cli.ts`
   - `src/options.ts`
   - focused unit tests
2. Implement a dependency-free CLI parser:
   - `--target-url <url>`
   - `--command <cmd>` or `-- <cmd...>`
   - `--host <host>`
   - `--port <port>`
   - `--ready-timeout <ms>`
   - `--config <path>` placeholder parsed but not fully expanded until Phase 93
3. Implement typed option normalization:
   - validate URLs and ports
   - reject target URL + command ambiguity unless the design explicitly supports both
   - default runtime host to `127.0.0.1`
   - default runtime port to an available documented value
4. Implement host config generation:
   - stable JSON-safe shape for browser host bootstrap
   - target URL is the canonical HMR path
   - runtime reload token / created timestamp for host-side reinit later
5. Add tests that assert:
   - target URL mode normalizes correctly
   - command mode preserves arbitrary framework commands without interpreting them
   - invalid URLs/ports fail with clear errors
   - generated host config contains no framework-specific assumptions
6. Add package docs stubs and a changeset for `@kehto/dev-runtime`.

## Out of Scope for This Plan

- Serving the browser host page.
- Actual iframe shell bootstrap.
- Full service wiring.
- Simulation UI controls.
- Playwright e2e proof.


## Completion Evidence

This plan is complete when the package builds, type-checks, has passing focused tests, and exposes both CLI and programmatic option surfaces ready for Phase 91.
