<diagnostic_protocol>
Per PITFALLS.md "three-question diagnostic" — when a spec fails:

1. **Is the dist newer than the source?**
   ```bash
   stat -c "%Y %n" tests/fixtures/napplets/nub-*/dist/index.html tests/fixtures/napplets/nub-*/src/main.ts | sort -n | tail -10
   ```
   If a `src/main.ts` mtime > corresponding `dist/index.html`, run `pnpm build` first.

2. **Was the console captured?**
   Read the Playwright report's first console error — that is the primary failure, not the Playwright timeout. `page.consoleMessages()` arrays in spec helpers are the canonical source.

3. **Was localStorage cleared?**
   `aclBeforeEach` calls `__clearLocalStorage__` — verify the spec is using `aclBeforeEach` and not bypassing it.

Additional Phase-21-specific diagnostic:

4. **Did the fixture finish AUTH?**
   `await waitForNappletReady(page, windowId)` — if this times out, the fixture's first SDK call is failing before AUTH completes. Check fixture's `init()` for unhandled rejections.

5. **Is `@napplet/core` deduplicated?**
   ```bash
   pnpm ls @napplet/core --depth 5 | head -20
   ```
   Per Pitfall 3 — if multiple instances appear, the `link:` overrides are not deduplicating. Run `pnpm install` to fix.
</diagnostic_protocol>
