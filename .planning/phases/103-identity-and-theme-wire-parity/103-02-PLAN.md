<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime request → reference service | An untrusted napplet-controlled message type and correlation id enter the known-action service dispatcher. |
| host signer/provider → identity result | Optional host callbacks can be absent, throw synchronously, reject asynchronously, or return data that must remain within the sanctioned result shape. |
| host theme input → stored state/callback | Host-supplied JavaScript values become napplet-visible request results and change notifications. |

## STRIDE Threat Register — ASVS L1

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-103-02-01 | Tampering | Identity/theme action dispatch | medium | mitigate | Use explicit known-action branches and return silently for every other type; test arbitrary and subscription-shaped inputs produce zero output. |
| T-103-02-02 | Information Disclosure | Identity provider failures | medium | mitigate | Preserve safe primary values and expose only stable operation-specific optional error text, never raw exceptions or stack data. |
| T-103-02-03 | Denial of Service | Signer/provider settlement | medium | mitigate | Normalize synchronous callbacks and Promises into one settlement path and test that every failure still yields exactly one sanctioned result. |
| T-103-02-04 | Tampering | Theme state ingress | medium | mitigate | Validate presence and string type of all required colors before storage; replace incomplete values with the fixed whole fallback before get or callback exposure. |
| T-103-02-05 | Repudiation | Read-only in-process service calls | low | accept | Durable audit logging is not warranted for correlated, non-mutating local reads; request-id and exact-cardinality tests provide deterministic evidence without collecting identity data. |
| T-103-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package install or dependency edit occurs; use only current workspace modules and pinned upstream text. |
