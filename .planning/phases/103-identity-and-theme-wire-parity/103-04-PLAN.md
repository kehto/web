<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Napplet JavaScript → injected namespace | Untrusted sandbox code can assign, delete, or redefine local namespace properties. |
| MessageEvent source → request correlation | Cross-window data must come from the trusted parent before it can settle a pending private-state read. |
| Parent change push → napplet callback | Automatic identity/theme changes cross into application-controlled handlers. |
| Other NAP domains → identity binding | Shell, intent, and INC payloads must not become alternate identity delivery paths. |
