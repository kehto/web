    ## Production Deployment Gap (C-05)

    **Kehto does NOT ship a production HTTP-header middleware.** The
    `serveNappletCsp` Vite plugin covers `pnpm dev` (port 5174) and
    `pnpm preview` / `pnpm test:serve:demo` (port 4174) — both hook into
    Vite's `configureServer` / `configurePreviewServer` lifecycle.

    For production deployments (static CDN, Express, Fastify, Cloudflare
    Worker, Nginx, etc.), **the host app is responsible** for configuring
    per-napplet CSP headers via whatever mechanism the deployment target
    supports. Possible approaches:

    - **Netlify**: `_headers` file with per-path rules.
    - **Vercel**: `vercel.json` `headers` array.
    - **Nginx**: `add_header Content-Security-Policy` in per-location blocks.
    - **Cloudflare Workers**: response-header rewriting middleware.
    - **Express/Fastify**: a per-request middleware reading from the
      `connectStore` equivalent (or a file-based sidecar) and emitting
      the header.

    The `connectStore` surface (`packages/shell/src/connect-store.ts`) is
    the authoritative source-of-truth in the browser. Production middleware
    must read from the same persistence layer (localStorage is browser-only;
    a production middleware needs a different sync mechanism — this is a
    known out-of-scope item for v1.7).

    ## Anti-features

    - **No `<meta http-equiv="Content-Security-Policy">`** anywhere in
      napplet HTML. Shell is sole CSP authority. CI-enforced via
      `pnpm audit:csp` (CONNECT-05 / C-03).
    - **No dismiss-as-implicit-allow** — dismiss (Escape or outside-click)
      and timeout both resolve as deny (M-04).
    - **No per-origin consent scopes** — origins are atomic; a grant is
      all-or-nothing per origin string (no read-only vs read-write
      granularity; deferred per 39-CONTEXT.md).
    - **No in-place CSP header mutation on revoke** — HTTP headers are
      immutable per response; iframe destroy+recreate is the only correct
      mechanism (C-04 hard requirement).

    import { test, expect } from '@playwright/test';

    /**
     * E2E-21 (v1.7 Phase 39 / Plan 39-05) — consent flow approve + dismiss=deny.
     *
     * Two tests:
     *   1. approve: __grantConnectOrigin__ test hook successfully records a grant;
     *      connectStore.check returns true via page.evaluate direct observation.
     *   2. dismiss = deny: without any grant in place, connectStore.check returns
     *      false AND the Vite middleware emits connect-src 'none' (D4 strict default).
     *      The modal UI dismiss flow is indirectly verified by the absence of grant
     *      (if dismiss=allow were the bug, any accidental trigger would leak).
     *
     * M-04 anti-feature: dismiss as implicit allow is explicitly forbidden.
     * This spec asserts the absence of leaked grants, complementing the per-file
     * grep in Plan 39-04's verify step (no resolve(true) on dismiss path).
     */
    test.describe('NUB-CONNECT consent flow (E2E-21)', () => {
      test.beforeEach(async ({ page }) => {
        await page.goto('/');
        // Reset: clear connect-store localStorage so tests start clean.
        await page.evaluate(() => {
          try { localStorage.removeItem('napplet:connect'); } catch {}
        });
        // Wait for demo to boot (config-demo is 11th — gate on its status sentinel).
        await page.waitForSelector('#config-demo-frame-container iframe', { timeout: 10_000 });
      });

      test('approve flow — __grantConnectOrigin__ records the grant and connectStore.check returns true', async ({ page }) => {
        // Arrange: no prior grants.
        const precheck = await page.evaluate(() => {
          // @ts-expect-error — test hook is attached at runtime, no static type.
          return window.__grantConnectOrigin__ !== undefined;
        });
        expect(precheck).toBe(true);

        // Act: grant wss://relay.damus.io to chat napplet.
        const granted = await page.evaluate(() => {
          // @ts-expect-error runtime hook
          return window.__grantConnectOrigin__('chat', '', 'wss://relay.damus.io');
        });
        expect(granted).toBe(true);

        // Assert: connectStore.check returns true via the demo's connectStore import.
        // The main.ts module re-exports it; we read via import('./shell-host.js') pattern
        // by directly consulting localStorage (persistence is synchronous inside grant()).
        const raw = await page.evaluate(() => localStorage.getItem('napplet:connect'));
        expect(raw).toBeTruthy();
        expect(raw!).toContain('chat:');
        expect(raw!).toContain('wss://relay.damus.io');
      });

      test('dismiss = deny — without any grant, CSP header is connect-src none and connectStore has no chat entry', async ({ page, request }) => {
        // Act: nothing — simulate user dismissing any consent (no grants stored).

        // Assert 1: localStorage has no connect-store entry, OR the chat entry is empty.
        const raw = await page.evaluate(() => localStorage.getItem('napplet:connect'));
        // raw may be null (never persisted) or an array with no chat entries.
        if (raw !== null) {
          expect(raw).not.toContain('chat:');
        }

        // Assert 2: the Vite middleware CSP header for chat is the strict default.
        const response = await request.get('/napplets/chat/index.html');
        expect(response.status()).toBe(200);
        const csp = response.headers()['content-security-policy'];
        expect(csp).toBeDefined();
        expect(csp).toMatch(/connect-src\s+'none'/);
      });
    });

     *   2. Fetch /napplets/chat/index.html; assert CSP includes wss://relay.test.
     *   3. Revoke all chat grants via __revokeConnect__ — dispatches shell:connect-revoked.
     *   4. Wait for main.ts to destroy + recreate the chat iframe.
     *   5. Fetch /napplets/chat/index.html AGAIN; assert CSP is connect-src 'none'.
     *
     * The key_link is the HTTP-header delivery mechanism: revoke mutates the
     * Vite middleware's in-memory Map (via fetch('/__connect-grants')); the
     * iframe reload triggers a fresh GET so the new header is delivered.
     */
    test.describe('NUB-CONNECT revocation (E2E-22 / CONNECT-04)', () => {
      test('revoke triggers iframe destroy+recreate and updated CSP excludes revoked origin', async ({ page, request }) => {
        await page.goto('/');
        await page.evaluate(() => { try { localStorage.removeItem('napplet:connect'); } catch {} });
        await page.waitForSelector('#chat-frame-container iframe', { timeout: 10_000 });

        // Grant
        const granted = await page.evaluate(() =>
          // @ts-expect-error runtime hook
          window.__grantConnectOrigin__('chat', '', 'wss://relay.test'),
        );
        expect(granted).toBe(true);

        // Wait for the sync POST to land on the Vite middleware (fire-and-forget fetch).
        // Poll the CSP header until it reflects the grant.
        await expect.poll(async () => {
          const r = await request.get('/napplets/chat/index.html');
          return r.headers()['content-security-policy'] ?? '';
        }, { timeout: 5_000 }).toMatch(/wss:\/\/relay\.test/);

        // Capture the iframe element before revoke so we can assert it was replaced.
        const iframeBefore = await page.$('#chat-frame-container iframe');
        expect(iframeBefore).not.toBeNull();

        // Revoke — dispatches shell:connect-revoked; main.ts destroys + recreates iframe.
        await page.evaluate(() =>
          // @ts-expect-error runtime hook
          window.__revokeConnect__('chat', ''),
        );

        // Wait for iframe destroy+recreate (new element replaces the old one in the DOM).
        await expect.poll(async () => {
          const current = await page.$('#chat-frame-container iframe');
          if (!current) return 'no-iframe';
          const isSame = await current.evaluate((el, old) => el === old, iframeBefore);
          return isSame ? 'same' : 'replaced';
        }, { timeout: 5_000 }).toBe('replaced');

        // Assert the refetched CSP header excludes the revoked origin.
        await expect.poll(async () => {
          const r = await request.get('/napplets/chat/index.html');
          return r.headers()['content-security-policy'] ?? '';
        }, { timeout: 5_000 }).toMatch(/connect-src\s+'none'/);
      });
    });
