# Plan 100-01: NAP-WEBRTC Runtime Parity

## Goal

Make `webrtc.*` work through Kehto's normal shell-owned service path and prove it in Paja and the playground.

## Requirements

WEBRTC-01 through WEBRTC-06.

## Steps

1. Add shell/runtime dispatch surface.
   - Add `WebrtcHooks` to `@kehto/shell`.
   - Advertise `webrtc` in `domains` and `naps` only when `hooks.webrtc.isAvailable()` is true.
   - Register `webrtc` in runtime NAP dispatch and service-only domain routing.

2. Add `@kehto/services` reference handler.
   - Export `createWebrtcService`.
   - Handle `open`, `send`, and `close`.
   - Return shaped upstream-compatible results for unavailable hooks, host exceptions, and unknown actions.
   - Provide a scoped event push helper so host code can emit `webrtc.event` to the requesting napplet.

3. Wire deterministic host implementations.
   - Add in-memory WebRTC controller to Paja.
   - Move `webrtc` from deferred to advertised/required Paja parity metadata.
   - Add playground demo WebRTC service and capability hook.

4. Add playground `webrtc-demo`.
   - Build a small napplet that waits for `webrtc`, opens a session, sends an opaque payload, observes a host-pushed message event, closes, and renders stable DOM sentinels.
   - Add gateway/guard/README updates.

5. Verify.
   - Focused unit tests for service, shell advertisement, runtime dispatch, and static guards.
   - Focused Playwright test for `webrtc-demo`.
   - Full repo gates, changeset, commit, push, PR stacked on `feat/nap-ble-parity`.

## Acceptance

- `webrtc` absent when unwired and present when wired.
- `webrtc.*` routes to a registered service through runtime.
- Playground `webrtc-demo` shows successful open/send/event/close through the real shell.
- Paja parity metadata has no deferred `webrtc` gap.
