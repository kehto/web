# Plan 96-01: NAP-COMMON Runtime Parity

## Goal

Add `NAP-COMMON` support end to end without implementing any other missing NAP domain.

## Cleanup Plan

This is additive protocol parity, not cleanup/refactor work. Keep edits localized to existing service-backed NAP extension points and delete no unrelated code.

## Tasks

1. Add shell/runtime contracts.
   - Add `CommonHooks` to `@kehto/shell`.
   - Advertise `common` in `buildShellCapabilities()` only when `hooks.common?.isAvailable()` is true.
   - Add shell-init tests for advertised, unavailable, and disabled-domain cases.
   - Add `common` to runtime domain handlers and NAP dispatch.

2. Add `@kehto/services` reference implementation.
   - Create `packages/services/src/common-service.ts`.
   - Export `createCommonService` and option/result types.
   - Handle `common.encodeNip19`, `common.decodeNip19`, `common.getProfile`, `common.follows`, `common.follow`, `common.unfollow`, `common.react`, and `common.report`.
   - Return shaped `*.result` envelopes with `ok: true` for deterministic/host-backed success and `ok: false` for denied or unavailable actions.
   - Add unit tests for all methods, host exceptions, default denial, and unknown actions.

3. Wire Paja.
   - Register `common` service in `createDevServices`.
   - Add `common` shell adapter hook when capability is enabled.
   - Add `common` to Paja parity metadata/service list.
   - Update parity tests.

4. Add playground napplet and host behavior.
   - Add `apps/playground/napplets/common-demo`.
   - Register it in `DEMO_NAPPLETS`.
   - Wire playground shell adapter/service for deterministic common behavior.
   - Add Playwright coverage for representative common results.

5. Release metadata and verification.
   - Add changesets for modified published packages.
   - Run focused unit tests first.
   - Run focused Playwright for `common-demo`.
   - Run full gates before PR: `pnpm build`, `pnpm type-check`, `pnpm test:unit`, `pnpm test:e2e`, docs check if docs changed, AI-slop scan, `git diff --check`.

## Stop Condition

`common` is proven with service/unit/static/Paja/playground/e2e evidence, no other missing NAP is implemented, and PR is open.
