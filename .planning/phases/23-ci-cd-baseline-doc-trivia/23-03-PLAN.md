    ```yaml
    # Phase 23 (v1.4): CI-03 requirement
    name: E2E

    on:
      push:
      pull_request:

    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    jobs:
      e2e:
        name: Playwright
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4

          - name: Setup pnpm
            uses: pnpm/action-setup@v3
            with:
              version: 10

          - name: Setup Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'
              cache: 'pnpm'

          - name: Install dependencies
            run: pnpm install --frozen-lockfile

          - name: Resolve Playwright version
            id: pw-version
            run: |
              VERSION=$(node -p "require('./node_modules/@playwright/test/package.json').version")
              echo "version=$VERSION" >> "$GITHUB_OUTPUT"

          - name: Cache Playwright browsers
            id: pw-cache
            uses: actions/cache@v4
            with:
              path: ~/.cache/ms-playwright
              key: ${{ runner.os }}-playwright-${{ steps.pw-version.outputs.version }}-${{ hashFiles('pnpm-lock.yaml') }}
              restore-keys: |
                ${{ runner.os }}-playwright-${{ steps.pw-version.outputs.version }}-
                ${{ runner.os }}-playwright-

          - name: Install Playwright browsers
            if: steps.pw-cache.outputs.cache-hit != 'true'
            run: pnpm exec playwright install --with-deps chromium

          - name: Install system dependencies (cache hit path)
            if: steps.pw-cache.outputs.cache-hit == 'true'
            run: pnpm exec playwright install-deps chromium

          - name: Run Playwright tests
            run: pnpm test:e2e
    ```

    Notes for the executor:
    - The `Resolve Playwright version` step reads the installed `@playwright/test` version (e.g., `1.54.0`) from `node_modules` AFTER `pnpm install` runs, then writes it to step output — this guarantees the cache key tracks the actual installed binary, not a stale lockfile entry.
    - The cache key has THREE components per CONTEXT.md decision: `runner.os` + Playwright version + `pnpm-lock.yaml` hash. The `restore-keys` fallback ladder allows partial cache reuse when only the lockfile hash changes (still need correct Playwright binaries).
    - On cache HIT, browser binaries are present but system libs (libnss3, libgtk-3, etc.) need installing — that's why `playwright install-deps chromium` runs in the cache-hit branch.
    - On cache MISS, `playwright install --with-deps chromium` installs both binaries and system deps in one step.
    - `pnpm test:e2e` runs `pnpm test:build` first (which is `pnpm build` — full turbo build), then `npx playwright test`. This satisfies CLAUDE.md's "Playwright only runs against built artifacts via `pnpm preview`" rule because the Playwright config spawns `pnpm preview` web servers (`:4173` harness, `:4174` demo). DO NOT add a separate `pnpm build` step — `test:e2e` includes it.
    - DO NOT add `--browser=chromium` overrides — Playwright config governs browser selection. We pre-install only chromium (matches CLAUDE.md: "this system uses Chromium via pacman"; same on `ubuntu-latest`).
    - DO NOT add report upload steps — out of scope for CI-03 baseline.
