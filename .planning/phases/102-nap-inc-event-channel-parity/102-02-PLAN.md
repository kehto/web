## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| channel.open target → session registry | Caller text attempts to select a trusted live endpoint. |
| channelId → internal channel map | Untrusted opaque identifiers address shell-owned membership state. |
| endpoint destruction → surviving peer | Lifecycle cleanup crosses from host state into a peer notification. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-102-04 | Spoofing | Channel target/peer | high | mitigate | Resolve only one live dTag, derive both opposite-peer fields from trusted sessions, enqueue target opened first, and reject window IDs, pubkeys, and duplicates. |
| T-102-05 | Elevation of Privilege | Channel membership | high | mitigate | Use shell-generated opaque IDs plus `peerOf` membership checks for every established-channel action. |
| T-102-06 | Denial of Service | Channel teardown indexes | medium | mitigate | Use one teardown primitive and assert both maps and all post-close routes are empty. |
| T-102-07 | Information Disclosure | Destroyed peer | medium | mitigate | Notify only surviving endpoints and remove routes before later traffic can deliver. |
| T-102-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package install occurs; use current workspace modules only. |
</threat_model>
