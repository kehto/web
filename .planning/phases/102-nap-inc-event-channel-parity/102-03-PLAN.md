## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| channel.open → ACL gate | Untrusted caller requests creation of persistent cross-napplet authority. |
| ACL mutation → live channel state | Host policy changes invalidate previously authorized membership. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-102-08 | Elevation of Privilege | `inc.channel.open` | high | mitigate | Retain the existing `relay:read` sender gate at open and prove denied opens create no state. |
| T-102-09 | Tampering | Established channel payload | high | mitigate | Remove per-message ACL/payload interpretation and rely only on opaque membership state. |
| T-102-10 | Elevation of Privilege | ACL revocation | high | mitigate | Observe block/relevant revoke synchronously and close all matching channels before more traffic is handled. |
| T-102-11 | Denial of Service | Unrelated ACL mutations | low | accept | Tests prove grant/unblock and unrelated revokes leave channels intact; residual risk is limited to explicit host policy mutation. |
| T-102-SC | Tampering | npm/pip/cargo installs | high | mitigate | No package install occurs and the external package gate remains intact. |
</threat_model>
