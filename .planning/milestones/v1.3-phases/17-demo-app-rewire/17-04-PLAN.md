<objective>
Rewire the demo's envelope-visibility surface (debugger tap, flow animator, sequence diagram, trace animator) to operate on NIP-5D envelope `type` strings instead of NIP-01 verbs + BusKind enum lookups. The core change is in `shell-host.ts` MessageTap: capture plain-object envelopes alongside the existing NIP-01 arrays, then drive all downstream rendering from the `envelopeType` field.

Purpose: Shell-bridge has been envelope-only since v1.2 (see shell-bridge.ts:211 — `if (typeof msg !== 'object' || msg === null || typeof msg.type !== 'string') return`), but the demo's tap still parses NIP-01 arrays. As a result the debugger shows `EVENT kind:29003 topic:shell:state-set` for what's actually a `storage.setItem` envelope. This plan closes that gap.

Output: The debugger's live log shows literal envelope `type` strings for every message the shell processes; flow-animator routes edges by domain prefix; sequence diagram and trace animator follow suit. Color modes still work.
</objective>
