  <name>Task 1: Install typedoc + create typedoc.json</name>
  <files>package.json, typedoc.json</files>
  <action>
    Addresses D-01 (DOCS-01) — typedoc@^0.28 at root devDep with `entryPointStrategy: "packages"`.

    Step 1: Install typedoc as a root (workspace-root) devDependency.
      Run: `pnpm add -D -w typedoc@^0.28`
      This adds `"typedoc": "^0.28.x"` to root `package.json` devDependencies and updates `pnpm-lock.yaml`.

    Step 2: Create `typedoc.json` at repo root with this EXACT content:
    ```json
    {
      "$schema": "https://typedoc.org/schema.json",
      "entryPointStrategy": "packages",
      "entryPoints": [
        "packages/acl",
        "packages/runtime",
        "packages/shell",
        "packages/services"
      ],
      "out": "docs/api",
      "exclude": ["**/*.test.ts", "**/dist/**", "**/node_modules/**"],
      "readme": "none",
      "includeVersion": true,
      "hideGenerator": true,
      "skipErrorChecking": true
    }
    ```

    Notes:
    - `entryPointStrategy: "packages"` instructs typedoc to treat each `entryPoints` element as a workspace package (reads its `package.json` → `main` or `src/index.ts`). Confirmed in STACK.md §4c.
    - `readme: "none"` avoids appending per-package README content into the API rendering — package READMEs are the human narrative, typedoc is the mechanical reference. They cross-link instead (Plan 22-02).
    - `skipErrorChecking: true` is intentional — typedoc runs against live source, and surface-level TS errors unrelated to JSDoc should not block doc generation. Real TS errors are caught by `pnpm type-check`.
    - `exclude` removes test files and build artifacts from the output graph.

    Do NOT install typedoc into a per-package devDep. It belongs at root only.
    Do NOT add typedoc to `turbo.json` — per STACK.md §4c, it's a manual dev-time tool, not a pipeline step.
  </action>
  <verify>
    <automated>grep -q '"typedoc"' package.json &amp;&amp; test -f typedoc.json &amp;&amp; node -e "const c=require('./typedoc.json'); if (c.entryPointStrategy !== 'packages') throw new Error('strategy mismatch'); if (c.entryPoints.length !== 4) throw new Error('entryPoints count !=4'); if (c.out !== 'docs/api') throw new Error('out mismatch'); console.log('ok');"</automated>
  </verify>
  <done>
