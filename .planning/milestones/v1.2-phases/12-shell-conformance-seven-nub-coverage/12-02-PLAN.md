<objective>
Rename sandbox-permission lookups in `shell.supports()` from bare names (e.g. `'popups'`) to the `perm:<permission>` namespace (e.g. `'perm:popups'`). NUB-capability lookups (e.g. `'relay'`, `'identity'`) retain the bare-name convention. Add a focused test that verifies both behaviors — sandbox lookups require `perm:` prefix, NUB lookups use bare names.

Purpose: Close DRIFT-SHELL-02 per canonical NIP-5D lines 81-94. The rename makes capability namespaces unambiguous — there is no longer any overlap between a NUB domain called "signer" and a sandbox permission called "signer" (the spec deltas call this out explicitly).

- `ShellCapabilities.sandbox` contract (JSDoc + any test fixtures) specifies entries MUST be `perm:` prefixed

The shell responds to `shell.ready` (see packages/shell/src/shell-bridge.ts lines 163-174)
with a `shell.init` envelope carrying `ShellCapabilities`. Napplets call
`shell.supports(cap)` (from @napplet/sdk/shim) which resolves locally against the
cached caps map. The CONTRACT being enforced by this plan:

```
shell.supports('popups')        -> false   (bare name, legacy, no longer recognized for sandbox)
shell.supports('perm:popups')   -> true    (namespaced, canonical)
shell.supports('relay')         -> true    (bare name, NUB lookup)
shell.supports('perm:relay')    -> false   (NUB capability does not use perm: prefix)
```

If the shell code does NOT currently ship a supports() helper (the napplet shim computes
locally against caps), this plan's test exercises the emitted capabilities directly:
- sandbox array entries all start with 'perm:'
- nubs array entries never start with 'perm:'

    ```ts
    import { describe, it, expect } from 'vitest';
    import { buildShellCapabilities } from '../src/shell-init.js';
    import type { ShellAdapter, ShellCapabilities } from '../src/types.js';

    function stubHooks(): ShellAdapter { /* minimal no-op hooks, relayPool present */ }

    /** Capability-lookup behavior a shell.supports() helper must enforce. */
    function lookup(caps: ShellCapabilities, capability: string): boolean {
      if (capability.startsWith('perm:')) return caps.sandbox.includes(capability);
      return caps.nubs.includes(capability);
    }

    describe('SH-C02: shell.supports() uses perm: namespace for sandbox permissions', () => {
      it('sandbox lookup requires the perm: prefix', () => {
        const caps: ShellCapabilities = { nubs: [], sandbox: ['perm:popups','perm:modals'] };
        expect(lookup(caps, 'perm:popups')).toBe(true);
        expect(lookup(caps, 'perm:modals')).toBe(true);
        expect(lookup(caps, 'popups')).toBe(false);       // bare name rejected
        expect(lookup(caps, 'modals')).toBe(false);
      });

      it('NUB lookup uses bare domain names, never perm:', () => {
        const caps: ShellCapabilities = { nubs: ['relay','identity','storage'], sandbox: [] };
        expect(lookup(caps, 'relay')).toBe(true);
        expect(lookup(caps, 'identity')).toBe(true);
        expect(lookup(caps, 'perm:relay')).toBe(false);   // nub cap does not use prefix
        expect(lookup(caps, 'perm:identity')).toBe(false);
      });

      it('buildShellCapabilities emits perm:-prefixed sandbox entries when provided', () => {
        // If buildShellCapabilities accepts a sandbox seed, test it here. Otherwise,
        // assert the empty-sandbox default and document the contract in the JSDoc check below.
        const caps = buildShellCapabilities(stubHooks());
        for (const entry of caps.sandbox) {
          expect(entry, `sandbox entry "${entry}" must start with 'perm:'`).toMatch(/^perm:/);
        }
      });

      it('ShellCapabilities JSDoc declares perm: contract', () => {
        const typesSource = require('node:fs').readFileSync(
          new URL('../src/types.ts', import.meta.url), 'utf8',
        );
        expect(typesSource).toMatch(/sandbox[\s\S]*perm:/);
      });
    });
    ```
