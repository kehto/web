<task type="auto"><name>2: Identity decrypt service handler</name>
<files>packages/services/src/identity-service.ts; packages/services/src/index.ts; packages/services/src/identity-service.test.ts</files>
<action>
- Extend `IdentityServiceOptions` with optional `getDecryptor` and `verifyEvent`.
- Export `HostDecryptBridge` / decrypt bridge types from `@kehto/services`.
- Add `identity.decrypt` switch branch.
- Verify the outer event before decrypting; failed verification or malformed event returns `malformed-wrap`.
- Auto-detect:
  - kind `4` => NIP-04.
  - kind `1059` => NIP-17 gift-wrap.
  - kind `14` OR base64/base64url content whose first decoded byte is `0x02` => NIP-44 direct.
  - otherwise return `unsupported-encryption`.
- Bridge calls:
  - NIP-04: `nip04Decrypt(event.pubkey, event.content)`.
  - NIP-44 direct: `nip44Decrypt(event.pubkey, event.content)`.
  - NIP-17: `unwrapGiftWrap(event)` then assert `seal.pubkey === rumor.pubkey`.
- Return `identity.decrypt.result` with `{ id, rumor, sender }`; for direct modes construct the rumor from the original event plus decrypted plaintext and never copy unrelated outer fields.
- Normalize bridge-thrown typed errors (`signer-denied`, `policy-denied`, etc.) into `identity.decrypt.error`; unknown bridge errors become `decrypt-failed`.
- Add focused tests for the three happy paths, all typed error codes, outer verify ordering, unsupported shape, missing bridge, impersonation mismatch, and id correlation.
</action>
