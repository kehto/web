## Decisions Made

- **pnpm version 10 (not 9 default):** Root `package.json` declares `packageManager: "pnpm@10.8.0"`. `pnpm install --frozen-lockfile` would fail in CI if the action installed pnpm 9 against a lockfile generated by pnpm 10 (lockfile schema mismatch). Pinned to `version: 10` in the action input.
- **No `paths-ignore` filter:** Per Phase 23 CONTEXT.md decision, all workflows run on every change. Cost is acceptable (~15-30s build on v1.3 baseline) and avoids the foot-gun of "docs-only" PRs silently skipping the gate.
- **`ubuntu-latest` only:** Multi-OS matrix is explicitly out of scope per v1.4 REQUIREMENTS.md.
- **`concurrency.cancel-in-progress: true`:** Stale runs on superseded commits are killed automatically — saves CI minutes and surfaces the latest result faster.
