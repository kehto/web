# Milestone v1.1: NIP-5D Migration Implementation

**Status:** ✅ SHIPPED 2026-04-07
**Phases:** 6-9
**Total Plans:** 8

## Overview

Implemented all code changes described in the v1.0 migration documents. All 4 kehto packages updated to support NIP-5D v0.1.0 envelope format. Clean break — no backward compatibility with NIP-01 array format.

## Milestone Summary

**Key Accomplishments:**
1. @kehto/acl: 2-segment identity keys, NUB domain resolution, localStorage migration utility (75 tests)
2. @kehto/runtime: NUB dispatch (envelope-only), AUTH machinery fully removed, 4 domain handlers, SessionEntry identity anchor (61 tests)
3. @kehto/shell: Envelope-only guard, window.nostr via postMessage handshake, shell.supports(), ACL migration trigger
4. @kehto/services: All 6 handlers migrated to NappletMessage envelope format (34 tests)

**Key Decisions:**
- Clean break — no backward compatibility, no dual-mode dispatch
- Dedicated `.error` type suffix for error responses
- `pubkey` field name for signer responses
- No ACL for read-only signer operations (getPublicKey, getRelays)
- postMessage handshake for window.nostr injection (Option B)

**Issues Resolved During Audit:**
- acl-state.ts getEntry() was using old 3-segment key format
- aclKey() in acl-store.ts was generating 3-segment keys
- 3 integration tests referenced removed sendChallenge
