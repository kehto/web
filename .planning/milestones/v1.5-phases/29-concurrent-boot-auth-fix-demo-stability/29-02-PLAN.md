(window as Window & { __grantMediaControl__?: () => boolean }).__grantMediaControl__ = (): boolean => {
  let mediaEntry: { windowId: string; info: NappletInfo } | null = null;
  for (const [windowId, info] of napplets.entries()) {
    if (info.name === 'media-controller') {
      mediaEntry = { windowId, info };
      break;
    }
  }
  if (!mediaEntry) {
    console.warn('[demo] __grantMediaControl__: media-controller napplet not loaded yet');
    return false;
  }
  if (!mediaEntry.info.authenticated) {
    console.warn('[demo] __grantMediaControl__: media-controller not yet authenticated');
    return false;
  }
  const pubkey = mediaEntry.info.pubkey ?? '';        // '' for NIP-5D napplets
  const dTag = mediaEntry.info.dTag ?? '';
  const hash = mediaEntry.info.aggregateHash ?? '';
  relay.runtime.aclState.grant(pubkey, dTag, hash, 'media:control');
  console.info('[demo] __grantMediaControl__: granted media:control to media-controller');
  return true;
};

Fix branch (i) auto-grant blueprint (apply ONLY if Task 1 routes here):

1. Inside `bootShell()`, after the existing `__grantMediaControl__` hook installation at shell-host.ts:903-925, add a post-mount auto-grant:
   - Use the napplets Map iterator from the `__grantMediaControl__` template.
   - Gate on `info.authenticated === true`.
   - Invoke via a one-shot `setTimeout` OR hook onto the tap.onMessage handler that already flips authenticated (preferred — event-driven, no arbitrary timeout).
   - Grant is idempotent per @kehto/acl — safe even if __grantMediaControl__ runs later in an E2E spec context.

2. Rationale for not removing `__grantMediaControl__`: Plan 27-04's E2E-13 spec invokes `window.__grantMediaControl__()` before asserting play/pause transitions. Removing the hook breaks the spec. Auto-grant + manual hook coexist — the manual hook returns true (already-granted = idempotent).

3. Rationale for event-driven over bootShell-synchronous: at bootShell return time, `loadNapplet('media-controller', ...)` has been called but the iframe's first ENVELOPE has not been received — Path B's authenticated flip has not happened. A synchronous grant call would find `info.authenticated === false` and bail (same as __grantMediaControl__'s existing bail path). The reliable hook is: inside the same tap.onMessage Path B handler that flips authenticated for media-controller, also invoke the grant. Alternative: polling retry every 200ms until authenticated, capped at e.g. 10 iterations (2s) — simpler but less elegant.
