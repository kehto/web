## How to read this document

Every drift item carries a stable ID of the form `DRIFT-<AREA>-<NN>` where `<AREA>` is one of
`ACL`, `RT` (runtime), `SHELL`, `SVC` (services), or `CORE` (dispatch / cross-cutting).
Every item has an owning package file, a one-line remediation, and a Target Phase mapping
(Phase 12, 13, or 14) so downstream planners can cite IDs directly.

Table shape for every drift section: six columns â€” ID, Drift Item, Current State, Spec/Package Requirement, Target Phase, Remediation Note.

## Scope of audit

- NIP-5D spec-level drift (protocol envelope shape, error suffix, dispatch contract).
- `@napplet/nub-*` message surface gaps for `ifc`, `relay`, `signer`, `storage`, `theme`.
- ACL capability-mapping gaps for the full nub surface.
- Dispatch/Core API drift (the hand-rolled `switch (domain)` in `runtime.ts` vs `createDispatch()` from `@napplet/core`).
